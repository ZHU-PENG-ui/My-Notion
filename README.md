My-Notion
基于 Next.js 14 + TypeScript 开发的仿 Notion 智能在线文档平台，专注于前端工程化与交互体验优化，实现了无限层级文档管理、实时富文本编辑、多语言适配等核心能力，解决传统笔记工具层级管理混乱、数据同步延迟高、国际化适配成本高等痛点，适用于个人知识沉淀与轻量团队协作。
核心功能

📂 无限层级文档树：支持文档的创建、移动、归档、恢复、批量删除，内置循环引用检测与孤儿节点自动提升机制，保障树形结构一致性

✍️ 高性能富文本编辑：基于 BlockNote 二次封装，支持图片拖拽上传 / 实时持久化，提供灵活的编辑器交互能力

🔄 实时数据同步：基于 Convex Subscription 实现数据变更实时推送，替代传统 REST 轮询，提升多端数据一致性

🌐 全链路国际化：支持英文 / 简体中文 / 繁体中文无缝切换，URL 级语言路由，三方组件与应用语言上下文同步

🎨 多端适配与主题：完成响应式布局开发，兼容 PC 端 / 移动端，支持亮色 / 暗色主题一键切换

🔒 用户认证：基于 Clerk 实现完善的用户身份认证体系，保障文档数据访问安全

📤 文件存储：集成 EdgeStore 实现图片文件的上传、存储与资源链接管理
核心技术亮点
1. 实时数据架构与文档树边界处理
基于 Convex Subscription 实现数据实时同步，替代 REST 轮询，降低服务端压力，提升数据一致性
文档树采用 parentDocument 自引用索引 + by_user_parent 复合索引设计，支持无限层级嵌套
实现祖先链遍历循环引用检测（移动操作）、递归级联归档 / 恢复、孤儿节点自动提升，覆盖所有树形结构操作的边界场景
2. 富文本编辑器工程化封装
对 BlockNote 进行二次封装，集成 EdgeStore 上传管道，实现图片拖拽上传与实时持久化
通过 forwardRef + useImperativeHandle 暴露命令式 API，支持父组件跨层级调度编辑器（如焦点控制），避免 props 传参导致的冗余渲染
编辑器内容通过 JSON 序列化 / 反序列化实现持久化，与 Convex 数据层无缝联动
3. 高性能状态管理
摒弃 React Context 全量重渲染问题，基于 Zustand 构建正交化多 Store（search/settings/coverImage/navigation），符合单一职责
利用 Zustand selector 细粒度订阅特性，实现字段级精准更新，有效消除非相关组件的冗余渲染
统一封装 ModalProvider，通过 isMounted 守卫解决 SSR 服务端渲染的 hydration mismatch 问题
4. 全链路国际化体系
基于 next-intl 实现 localePrefix: 'always' 模式的 URL 级语言路由，利于 SEO 且路由类型安全
设计编辑器字典动态映射层，确保 BlockNote 三方编辑器与应用全局语言上下文同步切换
多语言字典集中管理，支持快速扩展新的语言版本
5. 前端工程化最佳实践
基于 Next.js 14 App Router 实现服务端渲染 / 客户端组件按需拆分，提升首屏加载速度
全程使用 TypeScript 开发，完善的类型定义，降低开发阶段的类型错误
组件化、模块化设计，通用组件与业务组件解耦，提升代码复用性与可维护性
项目说明
本项目为前端全栈实践项目，专注于前端工程化、实时数据交互、富文本编辑、国际化等前端核心技术的落地与优化，所有功能均基于开源技术栈开发，无未实现的依赖与功能模块，可作为前端全栈学习、Notion 类产品开发的参考项目。
